# Glossário

## Router
Responsável por receber requisições HTTP e encaminhá-las para a camada responsável pela lógica de negócio.

## Service
Responsável por executar a lógica de negócio da aplicação.

## Schema
Contrato que define e valida os dados de entrada e saída da API.

## Lógica de Negócio
Conjunto de regras que definem o funcionamento do sistema.

## Baixo Acoplamento
Princípio em que os módulos possuem o mínimo de dependência entre si, facilitando manutenção e evolução do sistema.

## Configuração
Informação que altera o comportamento da aplicação sem modificar sua lógica.

## Context Manager
Estrutura da linguagem Python (`with`) que garante o gerenciamento correto de recursos.

## Resource
Qualquer objeto que precisa ser aberto e posteriormente liberado (arquivos, conexões com banco, clientes HTTP etc.).

## Resource Leak
Situação em que um recurso é aberto, mas nunca é liberado corretamente.

## Cache

Armazenamento temporário de informações em memória para evitar processamento ou leitura repetitiva, aumentando a performance da aplicação.

---

## Refactoring

Processo de reorganizar ou melhorar a estrutura interna do código sem alterar seu comportamento externo.

## Abstração
Princípio que consiste em esconder a implementação interna de uma funcionalidade, expondo apenas sua interface de uso.

## Parametrização

Técnica de tornar funções mais flexíveis permitindo receber valores externos em vez de utilizar valores fixos.

## Open/Closed Principle (OCP)

Princípio do SOLID que afirma que um software deve estar aberto para extensão e fechado para modificação.

## Escalabilidade

Capacidade de um sistema crescer mantendo desempenho, organização e facilidade de manutenção.

## Coesão

Princípio que busca manter funções, classes e módulos responsáveis por um único contexto de negócio.
